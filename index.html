Dynamic Programming:

Easy:

Hard:

  • Double Deck (NCPC 2024)
    Hint 1 This is the longest common subsequence problem but with extra restrictions on the input. Let f(x, y) be the solution of the first x numbers of the first list and the first y numbers of the second list. Then f(x, y) can be calculated only using f(x - 1, y), f(x, y - 1) and f(x - 1, y - 1). This is a well known dynamic programming solution for this problem. However it is to slow since it is Θ(n2). How does the extra restriction on the input affect the dynamic programming table.
    Hint 2 Note that most of the entries in the dynamic programming table are merely copies from their neighbours, the value can only increase in the N*K2 entries where the x'th number in the first list and the y'th number in the second list are equal.
    Hint 3 Only compute the entries in the dynamic programming table where the value can increase. How can this be calculated when we do not necessarily know the values of its neighbours.
    Hint 4 The value of f(x, y) if (x, y) is one of the points mentioned above can be calculated f(x, y) = 1 + maxi in {0, .., x - 1}, j in {0, .., y - 1}{f(i, j)}. This can be solved using a 2 dimentional range tree but that is both too complicated and too slow. Instead try to reduce this to a search in one dimension.
    Hint 5 Process the points by lowest x-value first then lowest y-value. Store each f(x, y) in a data structure keyed only on y. Answer each query on the datastructure as max of all keys < y.
    Hint 6 Use a segment tree or an order statistic binary search tree as the data structure needed in hint 5. (One can make one more observation and then use an array instead and binary search, which provides the same complexity but is practically faster and significantly easier to implement. The algorithm described by these hints is known as Hunt-Szymanski. The authors did not know/forgot about this algorithm, so you are not intended to know this when solving the problem.)

Very Hard:

  • Electronic Components (NCPC 2023)
    Hint 1 Let f(i, j) be the cost of placing the first i types of items where exactly j are placed by themselves.
    Hint 2 If you have written a correct transition function for the dp hinted above, you may have noticed that these can be written as a minimum over some parameter. Unfortunately the sequence we are doing minimum over is different for j for a fixed i for f(i, j). Do some math to fix this.
    Hint 3 For the math in the previous hint use a rewrite ala g(j) = f(i, j) + q(j), where q is some function such that the transition gets the form of a max over a sequence plus some constant for a fixed i, j pair.
    Hint 4 The transitions can be solved in logarithmic time with a segment tree. Beware of parity
